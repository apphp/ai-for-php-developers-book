# 2.3 Gradient descent on fingers

When we say that a model is learning, what we really mean is something very simple: it is reducing error step by step. Whether we are talking about linear regression, logistic regression, or a deep neural network, training is almost always driven by the same underlying mechanism.

That mechanism is called gradient descent.

{% hint style="info" %}
Gradient descent in its modern form did not emerge from a single paper. Its mathematical roots go back to the method of steepest descent, studied extensively by Augustin-Louis Cauchy in the 1840s. During the 20th century, the method evolved through optimization theory and numerical analysis before becoming one of the core tools of machine learning and neural networks.
{% endhint %}

The name may sound intimidating, but the idea itself is highly intuitive. It is based on the concept of a derivative: a derivative tells us the direction in which a function increases most rapidly. Therefore, if we want to reduce the value of a loss function, we need to move in the opposite direction.

That is exactly how gradient descent works: it tells us where to take the next step so that the error becomes smaller. It does this gradually, almost by feel – so intuitively that the entire process can be explained "on your fingers".

#### Error as a Landscape

Imagine a very simple situation. We have a model with a single parameter $$w$$. We change $$w$$ and calculate the error $$L(w)$$ each time. For example, the error could be MSE.

If we plot error as a function of $$w$$, we get a curve. In some places the error is larger, in others smaller, and at one point it reaches a minimum.

At this stage, it helps to switch abstractions. Stop thinking of error as a formula and start thinking of it as a landscape.

The parameter value $$w$$ represents your horizontal position. The error value $$L(w)$$ represents altitude.

It is as if we are walking through mountainous terrain and trying to reach the lowest point – the minimum.

#### Why It Is Called "Descent"

Imagine standing on the slope of a hill in dense fog. Obviously, you want to walk downhill. But you do not know where the lowest point is: there is no map, and you cannot see the horizon. All you have access to is the small patch of ground beneath your feet.

The only thing you can do is look around and figure out which direction slopes downward most steeply.

This is where the derivative comes in.

#### The Derivative as Slope

The derivative of the function $$L(w)$$ with respect to $$w$$ is essentially the slope of the curve at the current point.

Formally:

$$
L^′(w) = \frac{dL}{dw}
$$

But without the formal definition, it means the following:

* if the derivative is positive, increasing $$w$$ increases the error
* if the derivative is negative, increasing $$w$$ decreases the error
* if the derivative is zero, we are either at a minimum, a maximum, or a flat region (and in higher dimensions, possibly at a saddle point)

Geometrically, the derivative is the tangent of the angle formed by the tangent line to the graph.

#### Why the Derivative Gives the Direction of Motion

Suppose we are standing at the point $$w_0$$.

If \$$\frac{dL}{dw} > 0\$$, the graph goes "upward to the right". To move downhill, we need to move left, decreasing \$$w\$$.

If \$$\frac{dL}{dw} < 0\$$, the graph goes "downward to the right". In that case, it is beneficial to move right, increasing \$$w\$$.

Notice the key idea: we do not move _along_ the derivative – we move _against_ it.

That is why the gradient descent update step looks like this:

\$$\
w\_{new} = w\_{old} - \eta \frac{dL}{dw}\
\$$

Here \$$\eta\$$ is the learning rate.

It determines how large a step we take.

#### Intuition Behind the Step

You can read this formula almost literally:

"Take the current parameter value and move it in the direction opposite to the function's gradient at that point. Move by an amount proportional to the slope."

If the slope is steep, the derivative is large – so the step is larger.

If the slope is gentle, the derivative is small – so the step becomes smaller.

This is very similar to how a person carefully walks down a mountain. On a steep section, they take a noticeable step downward. On nearly flat ground, they move only slightly.

#### What Happens If the Step Is Too Large

Intuition might suggest speeding things up by choosing a very large \$$\eta\$$. But this leads to a classic problem.

If the step is too large, we stop descending and start jumping over the minimum. First we overshoot from the left, then from the right, then back again. The error no longer decreases – it oscillates.

On the other hand, if the step is too small, training becomes painfully slow. We are moving in the right direction, but so slowly that it seems as though the model is not learning at all.

This leads to one of the most important practical lessons: the learning rate is one of the most sensitive hyperparameters in model training.

If you would like an even more intuitive explanation, let's break it down "on our fingers".

Let's keep it extremely simple.

**1. First, the Derivative**

Imagine a hill.

A function is the shape of the hill.

A derivative tells you how steep the slope is at a specific point.

* if the derivative is positive → you are moving uphill
* if it is negative → you are moving downhill
* if it is 0 → you are at a peak or in a valley

\{% hint style="info" %\}\
In other words, a derivative tells you how quickly a function changes when a single variable changes.\
\{% endhint %\}

**2. Now the Gradient**

Now imagine you are no longer on a line but on a surface (like a 3D landscape).

For example, suppose you have a loss function \$$L(w₁, w₂)\$$.

This is no longer a curve but a landscape with hills and valleys.

A gradient is a vector (an arrow) that points in the direction of the steepest increase of the function.

It consists of partial derivatives:

\$$\
∇L = (\frac{dL}{dw\_1}, \frac{dL}{dw\_2})\
\$$

This means:

* \$$\frac{dL}{dw\_1}\$$ – how steep the surface is in the \$$w\_1\$$ direction
* \$$\frac{dL}{dw\_2}\$$ – how steep the surface is in the \$$w\_2\$$ direction

\{% hint style="info" %\}\
A gradient is a vector containing the partial derivatives of a function with respect to all of its variables.\
\{% endhint %\}

**3. The Connection (Most Important Part)**

The key thing to understand is that a derivative is simply a special case of a gradient.

* one variable → derivative
* many variables → gradient (a vector of derivatives)

**4. An Example Using a Loss Function**

Take a simple function:

\$$\
L(w) = (w - 3)^2\
\$$

Its graph looks like a bowl with a minimum at \$$w = 3\$$.

The derivative is:

\$$\
\frac{dL}{dw} = 2(w - 3)\
\$$

What does this mean?

* if \$$w > 3\$$ → the derivative is positive → move left
* if \$$w < 3\$$ → the derivative is negative → move right
* if \$$w = 3\$$ → the derivative is 0 → you are at the minimum

This is exactly what gradient descent uses.

\{% hint style="info" %\}\
Gradient descent moves in the direction opposite to the gradient in order to find the minimum of a function (the optimal parameters).\
\{% endhint %\}

**5. If There Are Multiple Parameters**

Now consider:

\$$\
L(w₁, w₂) = (w₁ - 3)² + (w₂ + 1)²\
\$$

The gradient is:

\$$\
∇L = (2(w₁ - 3), 2(w₂ + 1))\
\$$

This is the arrow that tells us:

* how to adjust \$$w\_1\$$
* how to adjust \$$w\_2\$$

**6. An Explanation for a Child**

* A derivative is "how steep it is forward or backward"
* A gradient is "an arrow in space showing where to go to climb fastest"
* Gradient descent is "walking in the opposite direction of that arrow to get downhill as fast as possible"

**In short**

* derivative = change along one axis
* gradient = changes along all axes at once
* gradient = a collection of derivatives

**Connecting It Back to the Theory**

Every training step follows the same rule:

\$$\
w = w - \eta \frac{dL}{dw}\
\$$

or

\$$\
w = w − learning\_rate \times derivative\
\$$

In other words:

* the derivative tells us "which way is uphill"
* we move in the opposite direction → downhill

And that is the entire idea behind gradient descent.

#### Many Parameters – Same Principle

So far we have discussed a single parameter \$$w\$$. In real models, there may be dozens, thousands, or even millions of parameters.

But the idea remains exactly the same.

The error becomes a function of many variables:

\$$\
L(w\_1, w\_2, \dots, w\_n)\
\$$

The derivative becomes a gradient – a vector of partial derivatives:

\$$\
\nabla L =\
\left(\
\frac{\partial L}{\partial w\_1},\
\frac{\partial L}{\partial w\_2},\
\dots,\
\frac{\partial L}{\partial w\_n}\
\right)\
\$$

This vector points in the direction of the _steepest increase_ of the loss function. Therefore, moving in the opposite direction gives the _steepest decrease_. This statement holds in the standard Euclidean space used by basic gradient descent algorithms.

Again, there is no magic here. We are simply walking downhill along the steepest slope.

#### Why Derivatives Are Essential

At this point it becomes clear why derivatives are not just an academic curiosity but a practical tool.

A derivative answers a simple question:

"If I change the parameters slightly, how will the error change?"

Gradient descent is nothing more than repeating the same cycle:

1. compute the current error
2. compute the derivatives
3. take a step that reduces the error

That cycle is what we call "training".

Before moving on to the implementation, let's quickly review:

The error is:

\$$\
error\_i = y\_i − \hat{y}\_i\
\$$

The loss function is:

\$$\
L = (y\_i - \hat{y}\_i)^2\
\$$

Derivatives:

With respect to \$$w\$$:

\$$\
\frac{\partial L}{\partial w} = -2 x\_i (y\_i - \hat{y}\_i)\
\$$

With respect to \$$b\$$:

\$$\
\frac{\partial L}{\partial b} = -2 (y\_i - \hat{y}\_i)\
\$$

#### PHP Implementation – From Scratch

Let's start with the simplest possible example: one feature, one weight. Predicting apartment prices based on floor area.

```php
// Обучающие данные
$x = [30, 40, 50, 60]; // площадь в м²
$y = [3, 4, 5, 6];     // цена (условно)

// Параметры модели
$w = 0.0; // вес
$b = 0.0; // смещение

// Параметры обучения
$learningRate = 0.0001;
$epochs = 5000;
$n = count($x);

// Градиентный спуск
for ($epoch = 0; $epoch < $epochs; $epoch++) {

    // Накопленные градиенты
    $dw = 0.0;
    $db = 0.0;

    // Проходим по всем точкам
    for ($i = 0; $i < $n; $i++) {
        // Предсказание модели
        $yPred = $w * $x[$i] + $b;
        
        // Ошибка предсказания
        // Если ошибка положительная – модель недооценила
        // Если отрицательная – переоценила
        $error = $y[$i] - $yPred;
    
        // Производные квадратичной ошибки
        // ∂L/∂w = −2 · xᵢ · (yᵢ − ŷᵢ)
        // где L = (yᵢ − ŷᵢ)², ŷᵢ = w · xᵢ + b
        $dw += -2 * $x[$i] * $error;        
        // ∂L/∂b = −2 · (yᵢ − ŷᵢ)
        // где L = (yᵢ − ŷᵢ)²
        $db += -2 * $error;
    }

    // Усредняем градиенты
    // Мы считаем средний градиент по всем точкам, а не делаем шаг после каждой. 
    // Это классический batch gradient descent.
    $dw /= $n;
    $db /= $n;

    // Обновляем параметры модели - шаг градиентного спуска
    // Мы двигаемся против направления градиента, потому что градиент указывает, куда ошибка растёт.
    // Маленький шаг – стабильное обучение.
    $w -= $learningRate * $dw;
    $b -= $learningRate * $db;
}

echo "w = {$w}, b = {$b}\n";

// Результат: w = 0.099958681685724, b = 0.0019740438781496
// Для этих данных результат будет близок к: y = 0.1x + 0.002
```

This code trains the simplest possible linear regression model using gradient descent.

The model has the form:

\$$\
y = w x + b\
\$$

where:

* \$$x\$$ – the feature (floor area),
* \$$y\$$ – the target value (price),
* \$$w\$$ – the weight (how much the price increases as the area increases),
* \$$b\$$ – the bias (base price).

The code:

1. takes a set of \$$(x, y)\$$ pairs,
2. starts with zero values for \$$w\$$ and \$$b\$$,
3. repeatedly computes the error over many epochs,
4. calculates how \$$w\$$ and \$$b\$$ should change to reduce that error,
5. gradually converges to values that best fit the data.

In the end, we obtain a line that lies as close as possible to the data points. The important thing is that there is no "ML magic" here. It is just a loop, some mathematics, and careful numerical updates.

#### Vectorized Version

When there are multiple features, it is more convenient to think in vectors.

```php
// Скалярное произведение двух векторов
// Используется для вычисления предсказания ŷ = w · x
function dot(array $a, array $b): float {
    $sum = 0.0;
    foreach ($a as $i => $v) {
        $sum += $v * $b[$i];
    }
    return $sum;
}

// Матрица признаков X
// Каждая строка — один объект
// Первый элемент — реальный признак (площадь)
// Второй элемент всегда равен 1 — это bias, включённый как признак
$X = [
    [30, 1],
    [40, 1],
    [50, 1],
    [60, 1],
];

// Истинные значения (целевая переменная)
$y = [3, 4, 5, 6];

// Вектор весов модели
// w[0] — вес площади
// w[1] — вес bias (смещение)
$w = [0.0, 0.0];

// Параметры обучения
$learningRate = 0.0001;
$epochs = 1000;
$n = count($X);

// Градиентный спуск
for ($epoch = 0; $epoch < $epochs; $epoch++) {

    // Вектор градиентов для каждого веса
    $dw = [0.0, 0.0];

    // Проходим по всем объектам
    for ($i = 0; $i < $n; $i++) {

        // Предсказание: скалярное произведение весов и признаков
        $yPred = dot($w, $X[$i]);

        // Ошибка модели на текущем объекте
        $error = $y[$i] - $yPred;

        // Обновляем градиенты по каждому весу
        // ∂L/∂w_j = -2 * x_j * (y - ŷ)
        foreach ($dw as $j => $_) {
            $dw[$j] += -2 * $X[$i][$j] * $error;
        }
    }

    // Обновляем веса, двигаясь против градиента
    foreach ($w as $j => $_) {
        $w[$j] -= $learningRate * ($dw[$j] / $n);
    }
}

// Итоговые веса модели
print_r($w);

// Результат: 
// Array
// (
//   [0] => 0.099956715379918
//   [1] => 0.0020679870606752
// )
// Для этих данных результат также будет близок к: y = 0.1x + 0.002
```

Here, the bias is already included as an additional feature with a value of 1. This is equivalent to the previous formula, just written in an expanded vector form.

This is a standard ML trick because it:

* works consistently across linear models, logistic regression, SVMs, and neural networks,
* simplifies backpropagation,
* allows everything to be stored in matrices,
* eliminates conditional branches from code and formulas.

In practice, almost all linear-model implementations in libraries work this way.

#### Summary

If we strip away the formulas, gradient descent is really about navigating an error landscape. The derivative is our compass. It does not tell us exactly where the global minimum is, but it always tells us which direction to move right now.

In that sense, machine learning is surprisingly similar to human experience. We rarely know the final destination in detail. But if we can tell what makes things worse and what makes them better, we can move in the right direction step by step.

That is exactly what gradient descent does.
