# 3.2 Logistic Regression

Linear regression works well for predicting numbers. But as soon as the task becomes binary – "yes or no", "spam or not spam", "customer will churn or stay" – it is no longer the right tool. The reason is that the model can produce values greater than 1 or less than 0 (that is, outside the range $$[0, 1]$$), making the result difficult to interpret.

Logistic regression solves exactly this problem. At its core, it is still a linear model, but instead of using the linear combination of features directly, we apply a nonlinear transformation – the [sigmoid function](https://en.wikipedia.org/wiki/Sigmoid_function). As a result, the model predicts not an arbitrary number, but the probability that an object belongs to class 1. This fits perfectly with the overall approach of this book: start with a simple linear idea, then carefully extend it to solve real-world problems.

#### From a linear model to probability

Let's begin with the familiar form of a linear model. Suppose we have an object described by features $$x$$. Then

$$
z = w \cdot x + b
$$

This is a standard [affine transformation](../getting-started/glossary.md#affine-transformation): the dot product of the feature vector and the weight vector plus a bias term. In linear regression, $$z$$ would be the final prediction. In logistic regression, however, $$z$$ is only an intermediate value. It is interpreted as the logarithm of the odds (log-odds) and is not itself a probability.

To convert the logit into a probability, we use the sigmoid (logistic) function:

$$
\sigma(z) = \frac{1}{1 + e^{-z}}
$$

It maps any real number to the interval $$(0, 1)$$. Large positive values of $$z$$ produce probabilities close to $$1$$, while large negative values produce probabilities close to $$0$$ .

One important idea is worth emphasizing here. Logistic regression is not simply "a classifier with a threshold". It is a probabilistic model from which a classifier is obtained by introducing a threshold. The model estimates the probability $$P(y = 1 \mid x)$$. Classification comes afterward as a practical decision based on a chosen probability threshold.

#### Decision boundary

Suppose we are solving a binary classification problem and decide that an object belongs to class \$$1\$$ if its predicted probability is at least \$$0.5\$$. Then the decision rule is

\$$\
\sigma(z) \ge 0.5\
\$$

Since \$$\sigma(0) = 0.5\$$, this is equivalent to the much simpler linear condition

\$$\
wx + b \ge 0\
\$$

This leads to an important conclusion: the decision boundary is the set of points for which

\$$\
wx + b = 0\
\$$

In two dimensions, this is a line. In three dimensions, it is a plane. In the general case, it is a hyperplane.

This clearly shows both the similarity to linear regression and the key difference. The decision boundary itself is linear, but the model's confidence changes nonlinearly as you move away from it. Near the boundary, the model is uncertain. Far from it, the model becomes increasingly confident in its prediction.

#### A bit of math: log-odds

Why do we use the sigmoid function? The answer lies in the concept of log-odds. Let \$$p = P(y = 1 \mid x)\$$. The odds are defined as

\$$\
\frac{p}{1 - p}\
\$$

Logistic regression assumes that the logarithm of these odds depends linearly on the features:

\$$\
\log\left(\frac{p}{1 - p}\right) = w x + b\
\$$

If we solve this equation for \$$p\$$, we obtain the sigmoid function once again. This is an important point: logistic regression is a linear model in the space of log-odds. The sigmoid is not an arbitrary trick – it is a direct consequence of assuming that the log-odds depend linearly on the input features.

#### Loss function and training

Since the model predicts probabilities, MSE is rarely used because it is not derived from the underlying probabilistic model and leads to less convenient optimization. Instead, logistic regression uses the logistic loss function, more commonly known as **log loss** or **binary cross-entropy**:

\$$\
L(y, p) = -\left\[ y \log(p) + (1 - y) \log(1 - p) \right]\
\$$

where \$$y \in {0,1}\$$ and \$$p = P(y = 1 \mid x)\$$.

For a dataset containing \$$n\$$ samples, the loss becomes

\$$\
L(\mathbf{w}) = -\frac{1}{n} \sum\_{i=1}^n \left\[ y\_i \log p\_i + (1 - y\_i)\log(1 - p\_i) \right], \quad p\_i = \sigma(x\_i^T \mathbf{w})\
\$$

The model is optimized using standard optimization methods – most commonly gradient descent. Mechanically, the process is very similar to linear regression, but because of the sigmoid function, the model becomes a nonlinear function of its parameters, and gradients must pass through this transformation. At the same time, in the standard formulation the loss function remains convex, guaranteeing a unique global minimum.

This is where logistic regression differs fundamentally from linear regression. In linear regression, we could derive a closed-form solution using the normal equations. That is not possible here. If we write down the gradient of the loss function,

\$$\
\nabla\_{\mathbf{w\}} L = \frac{1}{n} X^T (\mathbf{p} - \mathbf{y})\
\$$

and set it equal to zero, we obtain

\$$\
X^T\big(\sigma(X\mathbf{w}) - \mathbf{y}\big) = 0\
\$$

which is nonlinear with respect to \$$\mathbf{w}\$$. There is no way to solve this equation directly and express the weights explicitly in terms of the data.

> In other words, **logistic regression has no analytical solution based on the normal equations**.

Instead, training proceeds iteratively. We start with an initial set of weights and gradually improve them by minimizing the loss function. The simplest approach is gradient descent:

\$$\
\mathbf{w} := \mathbf{w} - \eta , \nabla\_{\mathbf{w\}} L\
\$$

where \$$\eta\$$ is the learning rate.

Intuitively, the process looks like this: the model makes a prediction, compares it with the correct answer, computes the error \$$\mathbf{p} - \mathbf{y}\$$, and slightly adjusts the weights in the direction that reduces this error. The process is repeated many times until the model converges.

Mechanically, this is very similar to linear regression. The main difference is that the sigmoid introduces a nonlinear relationship between the parameters and the predictions. Nevertheless, the loss function remains convex, so it still has a single global minimum – and gradient descent will converge to it.

**What this looks like in code**

If we ignore the matrix notation and focus on a single training example, the weight update is very straightforward. We compute the prediction, calculate the error, and make a small adjustment to the weights:

\$$\
w\_j := w\_j - \eta (p\_i - y\_i)x\_{ij}\
\$$

In code, it typically looks something like this:

```php
$z = dot($weights, $x) + $bias;
$p = sigmoid($z);

$error = $p - $y[$i];

foreach ($weights as $j => $w) {
    $weights[$j] -= $learningRate * $error * $x[$j];
}

$bias -= $learningRate * $error;
```

Here you can clearly see how the mathematical formula translates directly into just a few lines of code:

* compute the logit (`z`)
* pass it through the sigmoid → obtain the probability (`p`)
* compute the error (`p - y`)
* update the weights

This is gradient descent in its most basic form – gradually adjusting the model to fit the data one step at a time. We will use exactly this approach in the next practical example. In real applications, however, updates are usually performed on small batches of samples (mini-batches) rather than one sample at a time. This makes better use of vectorized operations and typically leads to faster, more stable convergence.

#### Case study: binary customer classification

Let's consider a simple yet realistic example. Imagine a subscription service. For each user, we know:

* the number of logins during the last month
* the average session duration
* the number of days since registration

Our goal is to predict whether the user will churn during the next month (\$$1\$$ – churn, \$$0\$$ – stay).

Logistic regression is particularly well suited for this task. The model returns the probability of churn. This is valuable not only from a technical perspective, but also from a product perspective: we can rank users by risk and focus our retention efforts only on those whose predicted probability exceeds a chosen threshold.

It is important to note that choosing the threshold \$$(0.5, 0.7, 0.9)\$$ is a business decision. The model provides a probability, not a hard yes-or-no answer.

#### Limitations of logistic regression

Although the sigmoid introduces nonlinearity in the probability space, logistic regression remains a linear classifier in terms of its decision boundary. If the classes are not linearly separable, logistic regression cannot perfectly separate them with a single hyperplane. Instead, it produces the best linear approximation it can. In such cases, the solution may be to engineer better features or use more powerful models, such as decision trees, kernel SVMs, or neural networks.

Despite these limitations, logistic regression remains one of the most widely used models in applied machine learning. It is simple, interpretable, robust, and often serves as an excellent starting point for solving classification problems.

#### Summary

Logistic regression is a natural extension of linear models to classification tasks. It preserves the simplicity of the linear approach while adding a probabilistic interpretation, a well-defined decision boundary, and a direct connection to real-world business problems. That is why it is almost always the first model people learn when studying binary classification in machine learning.
