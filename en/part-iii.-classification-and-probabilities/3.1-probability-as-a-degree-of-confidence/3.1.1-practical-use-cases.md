# 3.1.1 Practical use cases

This chapter is deliberately not about training models, choosing algorithms, or fine-tuning hyperparameters. We'll assume that our model already exists and is capable of producing probabilistic predictions. Our focus is not on how the model was built, but on what to do with the probabilities it returns.

The case studies in this chapter all illustrate the same idea from different perspectives: probability becomes truly useful after a model has made its prediction—in the interpretation of results, decision-making, and updating confidence as new information becomes available.

We will treat probability not as an abstract number between 0 and 1, nor as the model's subjective "confidence," but as a practical tool embedded in the decision-making process. In one scenario, probability helps us choose the action with the lowest expected risk. In another, it helps explain the model's output to a human. In a third, it allows us to update an existing belief when new evidence arrives.

To make the concepts as clear as possible, the examples are implemented in two ways:

* some case studies are written in pure PHP, without using any machine learning libraries;
* others use Rubix ML, demonstrating how the same ideas appear in real machine learning code rather than only in theory.

At the same time, several guiding principles should be kept in mind. Rubix ML is used only where it helps us focus on probabilistic reasoning rather than implementation details. The code is intentionally written so that it can be read and understood without extensive knowledge of the library. Each case study concludes with a key takeaway, explaining what probability contributes in that particular scenario and why simply choosing the class with the highest probability is often not enough.
