# 5.6.2 Практические кейсы

Теория Named Entity Recognition может показаться простой: есть текст, есть сущности, есть метки. Однако именно на этом уровне становится видно, как AI превращает неструктурированный текст в данные, с которыми уже может работать обычное приложение.

В реальных системах NER редко используется отдельно. Он становится частью более крупных процессов: автоматической обработки документов, CRM-систем, финансового анализа, поиска информации и подготовки данных для RAG-архитектур.

В этой части главы мы рассмотрим несколько практических кейсов на PHP. Мы разберём, как извлекать сущности из текста, как сохранять результаты в структуру приложения, где NER помогает заменить ручную обработку и как адаптировать модель под собственный домен.

Для примеров будем использовать локальный inference через библиотеку `mitie-php`, которая позволяет запускать NER-модели непосредственно из PHP-приложения. Это покажет важный инженерный подход: AI-модель становится частью обычного backend-приложения, а не отдельным “магическим” сервисом. Кроме этого рассмотим также несколько других подходов, таких как использование TransformersPHP.

Отдельно рассмотрим процесс дообучения NER-модели. В реальных проектах стандартные модели часто знают общие сущности, но не понимают внутренние термины бизнеса: номера договоров, артикулы товаров, внутренние идентификаторы или специализированные категории данных.

К концу этого раздела вы сможете построить полноценный pipeline:

```
текст → извлечение сущностей → структурированные данные → бизнес-логика приложения
```

Итак, мы с вами рассмотрим следующие кейсы:

* **Кейс 1. Автоматический разбор договора (Legal / CRM)**\
  Извлекаем стороны, даты и суммы из юридических документов с помощью NER.
* **Кейс 2. Финтех – автоматический анализ транзакционных сообщений**\
  Находим суммы, получателей и важные параметры в транзакционных сообщениях.
* **Кейс 3. Автоматическое построение базы контактов из email**\
  Автоматически строим базу контактов из переписки и извлекаем организации, людей и места.
* **Кейс 4. Подготовка данных для RAG**\
  Используем сущности для индексации документов и улучшения поиска по базе знаний.
* **Кейс 5. Дообучение NER-модели (на примере MITIE)**\
  Дообучаем модель MITIE на собственных данных и добавляем новые типы сущностей.
