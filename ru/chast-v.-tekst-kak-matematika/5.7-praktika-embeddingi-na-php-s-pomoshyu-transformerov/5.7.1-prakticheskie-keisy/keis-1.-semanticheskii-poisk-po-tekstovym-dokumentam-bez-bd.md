# Кейс 1. Семантический поиск по текстовым документам (без БД)

#### Цель кейса

В этом кейсе мы создадим минимальную версию семантического поиска на PHP.

Наша задача – научиться искать документы не по совпадению слов, а по смысловой близости.

Для этого мы возьмём обычную коллекцию текстовых файлов, преобразуем их содержимое в эмбеддинги с помощью трансформер-модели и построим простой индекс на основе JSON.

Этот пример специально сделан без базы данных и внешних сервисов. Его цель – показать саму механику работы:

```
текст
  ↓
эмбеддинг
  ↓
вектор
  ↓
сравнение
  ↓
релевантный результат
```

После понимания этого цикла переход к векторной базе данных или RAG становится уже инженерной задачей, а не магией.

#### Сценарий

Представим, что у нас есть папка с документацией проекта:

```
documents/

php-ai.md
transformers.md
laravel.md
machine-learning.txt
```

Каждый документ содержит обычный текст.

Например:

`php-ai.md`

```
PHP постепенно становится частью AI-инфраструктуры.
Разработчики используют PHP для создания приложений
с языковыми моделями и интеллектуальными функциями.
```

`transformers.md`

```
Трансформеры позволяют создавать модели,
которые понимают контекст текста и строят
семантические представления данных.
```

Теперь пользователь вводит запрос:

```
как добавить искусственный интеллект в PHP приложение
```

Классический поиск по словам может не найти ничего.

Почему?

Потому что в документе может не быть слов:

```
искусственный интеллект
```

Вместо этого используются:

```
AI-инфраструктура
языковые модели
интеллектуальные функции
```

Для человека эти выражения близки по смыслу.

Для обычного поиска – разные строки.&#x20;

То есть, классический поиск по ключевым словам может не распознать смысловую близость документов. Если совпадает лишь небольшая часть терминов, такой документ может получить низкий рейтинг или вообще не попасть в результаты поиска.

#### Идея решения

Вместо сравнения слов мы отображаем текст в многомерное векторное пространство (пространство эмбеддингов).

Каждый документ превращается в вектор:

```
Документ

"PHP постепенно становится частью AI-инфраструктуры..."
        ↓
[0.021, -0.043, 0.112, ...]
```

Этот вектор называется эмбеддингом. Он представляет собой числовое описание смысла текста, которое позволяет сравнивать документы по семантической близости.

То же самое делаем с запросом пользователя:

```
"Как добавить искусственный интеллект в PHP приложение?"
        ↓
[0.018, -0.039, 0.118, ...]
```

Теперь задача становится простой:

> найти документы, чей вектор ближе всего к вектору запроса.

Для этого будем использовать косинусное сходство.

#### Что мы демонстрируем

Этот кейс показывает базовый инженерный цикл работы с эмбеддингами:

**1. Индексация**

Документы заранее преобразуются в векторы:

```
documents
    ↓
TransformersPHP
    ↓
embeddings
    ↓
JSON index
```

Это происходит один раз.

**2. Поиск**

Каждый пользовательский запрос проходит через ту же модель:

```
query
    ↓
TransformersPHP
    ↓
query embedding
    ↓
similarity search
```

**3. Выбор результата**

После получения оценки похожести:

```
document A → 0.84
document B → 0.61
document C → 0.32
```

мы сортируем результаты и возвращаем top-N документов.

#### Почему начинаем именно с этого кейса

Это самый простой способ увидеть эмбеддинги в действии.

Нам не нужны:

* базы данных
* vector store
* сложная инфраструктура
* обучение моделей

Достаточно: PHP, TransformersPHP, готовой модели и нескольких математических операций.

Именно этот принцип – представление текста в виде эмбеддингов и поиск ближайших векторов – лежит в основе большинства современных систем семантического поиска и RAG (об этом мы будем говорить в следующей главе).

#### Технический стек

В этом примере используем:

**PHP**

Отвечает за:

* загрузку документов
* управление процессом поиска
* сортировку результатов
* бизнес-логику

**TransformersPHP**

Отвечает за:

* токенизацию текста
* запуск трансформер-модели
* получение эмбеддингов

Пример:

```php
use function Codewithkyrian\Transformers\Pipelines\pipeline;

$embedder = pipeline(
    task: 'embeddings',
    modelName: 'Xenova/paraphrase-multilingual-MiniLM-L12-v2'
);
```

#### Реализация

Получаем эмбеддинг текста:

```php
$text = 'Как добавить искусственный интеллект в PHP приложение?';

$result = $embedder($text, normalize: true, pooling: 'mean');
$embedding = array_map(static fn ($v): float => (float) $v, $result[0]);

print_r($embedding);
```

На выходе получаем массив чисел:

```
[0.021, -0.043, 0.112, ...]
```

Это и есть цифровое представление смысла текста.

#### Индексация документов

Проходим по файлам:

```php
$files = glob(__DIR__ . '/documents/*');

$documents = [];

foreach ($files as $file) {
    $text = file_get_contents($file);
    $result = $embedder($text, normalize: true, pooling: 'mean');
    $embedding = array_map(static fn ($v): float => (float) $v, $result[0]);
    
    $documents[] = [
        'id' => basename($file),
        'text' => $text,
        'embedding' => $embedding,
    ];
}
```

Сохраняем полученные эмбеддинги в JSON-файл, который будет использоваться как простой поисковый индекс:

```php
file_put_contents('embeddings.json', json_encode($documents));
```

#### Поиск

Получаем запрос:

```php
$query = 'Как добавить искусственный интеллект в PHP приложение?';
```

Создаём его эмбеддинг:

```php
$result = $embedder($query, normalize: true, pooling: 'mean');
$queryEmbedding = $result[0];
```

Теперь сравниваем с документами.

#### Cosine similarity

Для нормализованных векторов скалярное произведение эквивалентно cosine similarity, поэтому отдельно вычислять длину векторов уже не требуется.

```php
function cosineSimilarity(array $a, array $b): float {
    return array_sum(
        array_map(fn($x, $y) => $x * $y, $a, $b)
    );
}
```

#### Результат

Запрос:

```
Как добавить искусственный интеллект в PHP приложение?
```

Например, система может вернуть:

```
0.84 php-ai.md
PHP постепенно становится частью AI-инфраструктуры.
Разработчики используют PHP для создания приложений
с языковыми моделями и интеллектуальными функциями.

0.62 transformers.md
Трансформеры позволяют создавать модели,
которые понимают контекст текста.

0.38 laravel.md
Laravel используется для создания
современных веб-приложений.
```

Система нашла правильный документ, хотя слова запроса и документа отличаются.

#### Где применять в production

Этот подход можно использовать для:

* поиска по документации
* корпоративных баз знаний
* юридических документов
* внутренних Wiki
* архивов сообщений
* технических материалов

Например:

Вместо:

```
найти OAuth
```

пользователь пишет:

```
как подключить авторизацию через внешний сервис
```

и получает релевантный результат.

#### Ограничения

Семантический поиск не заменяет все остальные виды поиска.

Есть ситуации, где обычный индекс лучше:

* точные идентификаторы
* номера заказов
* артикулы
* коды ошибок

Также нужно учитывать:

* размер модели
* время инференса (получения эмбеддингов)
* качество выбранной модели
* размер коллекции документов

Для небольших коллекций документов хранения эмбеддингов в JSON-файле обычно достаточно. Однако поиск остаётся линейным, поэтому при росте числа документов обычно переходят к специализированным векторным индексам (vector store).

#### Выводы

В этом кейсе мы построили минимальную семантическую поисковую систему:

1. Получили документы
2. Создали эмбеддинги через TransformersPHP
3. Сохранили векторный индекс
4. Преобразовали запрос пользователя в вектор
5. Нашли документы с наиболее близкими эмбеддингами, соответствующими смысловой близости.

Главный вывод:

> эмбеддинги позволяют добавить в PHP-приложение новый тип поиска – не по словам, а по смыслу.

Именно этот фундамент дальше используется в более сложных системах: RAG, AI-ассистентах, рекомендательных механизмах и агентных архитектурах.
