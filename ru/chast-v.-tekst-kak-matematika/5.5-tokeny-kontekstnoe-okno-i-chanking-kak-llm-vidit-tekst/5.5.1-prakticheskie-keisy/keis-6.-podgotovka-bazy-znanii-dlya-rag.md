# Кейс 6. Подготовка базы знаний для RAG

Это лучший завершающий кейс главы.

#### Цель

Подготовить набор документов к загрузке в векторную базу.

#### Архитектура

```
Markdown документы
          ↓
     Tiktoken
          ↓
 Token-based Chunking
          ↓
     Overlap
          ↓
    Embeddings
          ↓
   Vector Database
```

#### Реализация

На этапе этой главы мы выполняем только первые три шага:

1. читаем документы;
2. считаем токены;
3. создаём чанки;
4. добавляем overlap;
5. сохраняем результат в JSON.

```php
[
    {
        "id": 1,
        "text": "...",
        "tokens": 782
    },
    {
        "id": 2,
        "text": "...",
        "tokens": 799
    }
]
```
