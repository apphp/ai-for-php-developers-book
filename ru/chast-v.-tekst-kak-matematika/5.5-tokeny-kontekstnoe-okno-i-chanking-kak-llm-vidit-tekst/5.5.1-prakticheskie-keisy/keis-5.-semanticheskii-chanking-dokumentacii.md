# Кейс 5. Семантический чанкинг документации

#### Цель

Сравнить два подхода к разбиению текста.

#### Вариант 1

По токенам:

```
800 токенов
800 токенов
800 токенов
```

#### Вариант 2

По структуре документа:

```
Глава "Авторизация"

Глава "REST API"

Глава "Ошибки"
```

#### Реализация

Для Markdown:

```php
<?php

$sections = preg_split(
    '/^#{1,2}\s+/m',
    $markdown
);
```

После чего размер каждого раздела проверяется через Tiktoken.

#### Что изучаем

* semantic chunking;
* структуру документа;
* качество retrieval.
