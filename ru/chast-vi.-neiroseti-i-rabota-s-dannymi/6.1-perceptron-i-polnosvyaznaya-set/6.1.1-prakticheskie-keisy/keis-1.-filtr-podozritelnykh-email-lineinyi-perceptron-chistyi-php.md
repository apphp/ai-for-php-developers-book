# Кейс 1. Фильтр подозрительных email – линейный перцептрон (чистый PHP)

#### Цель кейса

Разработать простой классификатор, который автоматически определяет, является ли входящее письмо потенциально фишинговым. В качестве модели используется линейный перцептрон, полностью реализованный на PHP без сторонних библиотек машинного обучения.

Этот кейс демонстрирует, что даже самый простой алгоритм машинного обучения способен решать практические задачи информационной безопасности и может служить базовым уровнем защиты в корпоративных системах.

#### Практическая задача

Практически каждая компания сталкивается с фишинговыми письмами:

* “Ваш пароль скоро истечет”;
* “Подтвердите учетную запись”;
* “Получите срочную выплату”;
* “Обновите банковские данные”.

Подобные сообщения имеют характерные признаки:

* большое количество ссылок;
* слова, оказывающие психологическое давление;
* использование внешних доменов;
* небольшую или, наоборот, чрезмерно большую длину текста.

Даже если использовать всего несколько простых признаков, многие фишинговые письма уже можно отделить от обычной корпоративной переписки.

Именно эту задачу и будет решать наш первый классификатор.

***

#### Почему именно перцептрон?

Линейный перцептрон является одной из самых простых моделей бинарной классификации.

Если два класса объектов можно разделить одной гиперплоскостью, то перцептрон способен найти эту разделяющую границу.

Для нашей задачи это означает следующее:

* безопасные письма находятся по одну сторону границы;
* подозрительные — по другую.

Именно поэтому перцептрон часто используется как первая модель при построении систем машинного обучения.

***

#### Какие признаки будем использовать?

Предположим, что перед обучением мы уже извлекли из каждого письма несколько числовых характеристик.

| Признак | Описание                                                                   |
| ------- | -------------------------------------------------------------------------- |
| x₁      | количество ссылок в письме                                                 |
| x₂      | количество слов-триггеров (“urgent”, “verify”, “password”, “login” и т.п.) |
| x₃      | наличие внешнего домена (0 или 1)                                          |
| x₄      | длина письма (например, количество слов)                                   |

После извлечения признаков каждое письмо превращается в обычный числовой вектор.

Например:

```
[4, 3, 1, 120]
```

или

```
[0, 0, 0, 560]
```

Именно такие векторы и будут поступать на вход нашей модели.

***

#### Подготовим обучающую выборку

Для примера возьмем небольшой искусственный набор данных.

```php
$X = [
    [5, 4, 1, 80],
    [4, 3, 1, 120],
    [0, 0, 0, 600],
    [1, 0, 0, 420],
    [6, 5, 1, 60],
    [0, 1, 0, 500],
];

$y = [
    1,
    1,
    0,
    0,
    1,
    0
];
```

Здесь:

* 1 — фишинговое письмо;
* 0 — обычное письмо.

Разумеется, в реальном проекте обучающая выборка должна содержать тысячи или десятки тысяч примеров, однако для демонстрации алгоритма этого вполне достаточно.

***

#### Реализация линейного перцептрона

Ниже приведена минимальная реализация модели.

```php
<?php

class Perceptron
{
    private array $w;
    private float $b = 0.0;
    private float $lr;

    public function __construct(int $nFeatures, float $lr = 0.1)
    {
        $this->w = array_fill(0, $nFeatures, 0.0);
        $this->lr = $lr;
    }

    public function predict(array $x): int
    {
        $z = $this->b;

        foreach ($x as $i => $value) {
            $z += $this->w[$i] * $value;
        }

        return $z > 0 ? 1 : 0;
    }

    public function train(array $X, array $y, int $epochs = 50): void
    {
        for ($e = 0; $e < $epochs; $e++) {

            foreach ($X as $i => $x) {

                $pred = $this->predict($x);

                $error = $y[$i] - $pred;

                foreach ($x as $j => $value) {
                    $this->w[$j] += $this->lr * $error * $value;
                }

                $this->b += $this->lr * $error;
            }
        }
    }
}
```

***

#### Обучение модели

Создадим объект и обучим его на нашей выборке.

```php
$model = new Perceptron(4);

$model->train($X, $y);
```

После обучения веса модели будут отражать важность каждого признака.

***

#### Проверка на новом письме

Предположим, пришло новое сообщение со следующими характеристиками:

* 4 ссылки;
* 3 слова-триггера;
* есть внешний домен;
* длина письма — 90 слов.

Подготовим вектор признаков.

```php
$email = [4, 3, 1, 90];

$result = $model->predict($email);

echo $result ? 'Фишинговое письмо' : 'Безопасное письмо';
```

В результате программа выведет:

```
Фишинговое письмо
```

***

#### Что происходит внутри модели?

Во время обучения перцептрон постепенно изменяет веса признаков.

Если модель ошиблась:

* веса увеличиваются;
* либо уменьшаются,

чтобы в следующий раз разделяющая гиперплоскость проходила более удачно.

После завершения обучения каждый признак получает собственный коэффициент.

Например:

```
Количество ссылок      → +1.8
Слова-триггеры         → +2.4
Внешний домен          → +3.1
Длина письма           → -0.2
```

Из этих коэффициентов уже можно сделать выводы.

Например:

* наличие внешнего домена значительно повышает вероятность фишинга;
* большое количество слов-триггеров также является сильным индикатором;
* длина письма практически не влияет на решение модели.

Именно поэтому перцептрон считается одной из самых интерпретируемых моделей машинного обучения.

***

#### Где такой подход применяется на практике?

Несмотря на простоту, подобные модели до сих пор используются в промышленной разработке как первый уровень фильтрации.

Типичные сценарии применения:

* предварительная фильтрация входящей почты;
* оценка риска открытия письма пользователем;
* приоритизация подозрительных сообщений для дальнейшей проверки;
* быстрый скоринг событий безопасности;
* фильтрация объектов перед передачей в более сложную модель машинного обучения.

Во многих системах такой классификатор используется как быстрый фильтр, а более дорогие модели (логистическая регрессия, SVM, градиентный бустинг или нейронные сети) анализируют только подозрительные сообщения.

***

#### Ограничения модели

Несмотря на простоту реализации, линейный перцептрон имеет ряд ограничений.

Главное из них — модель способна разделять только линейно разделимые данные. Если признаки двух классов сильно пересекаются, перцептрон не сможет найти разделяющую гиперплоскость, которая корректно классифицирует все объекты.

Кроме того, качество классификации напрямую зависит от выбранных признаков. Если использовать недостаточно информативные характеристики письма, модель будет ошибаться независимо от алгоритма обучения.

Поэтому в реальных системах большое внимание уделяется этапу извлечения признаков (Feature Engineering), а сам перцептрон часто используется как отправная точка перед переходом к более сложным моделям.

***

#### Итоги

В этом кейсе мы построили полноценный бинарный классификатор фишинговых писем, используя только возможности языка PHP. Несмотря на минимальный объем кода, модель уже способна автоматически отделять потенциально опасные письма от обычной корпоративной переписки на основе набора простых числовых признаков.

Главные преимущества такого решения:

* полностью реализуется на чистом PHP без сторонних ML-библиотек;
* очень быстро обучается и выполняет предсказания;
* легко интегрируется в существующие корпоративные приложения;
* обладает высокой интерпретируемостью — вклад каждого признака можно объяснить;
* является отличной отправной точкой для изучения задач бинарной классификации.

Для B2B-продуктов в области Security Awareness, симуляции фишинговых атак и анализа корпоративной почты такой перцептрон представляет собой хороший baseline-модуль. Он позволяет быстро внедрить интеллектуальную фильтрацию и создать основу для последующего перехода к более сложным моделям машинного обучения, таким как логистическая регрессия, деревья решений или нейронные сети.
