# Кейс 5. Обучение модели как минимизация ошибки

#### Идея кейса

До этого момента мы рассматривали loss как способ оценить модель.

Теперь важно показать главное:

> Обучение модели – это и есть процесс минимизации loss-функции.

В этом кейсе мы буквально "увидим" обучение, не используя ни градиентный спуск, ни магию библиотек.

#### Цель кейса

* Интуитивно связать обучение и минимизацию ошибки
* Показать, что модель – это просто набор параметров
* Наглядно продемонстрировать: найти лучшие параметры = найти минимум loss

#### Сценарий

У нас есть простая зависимость: $$y = 2x$$

Данные:

```php
$x = [1, 2, 3, 4];
$y = [2, 4, 6, 8];
```

Мы обучаем линейную модель (без свободного члена): $$\hat{y} = w x$$

Наша задача – найти такое значение $$w$$, при котором $$MSE(w)$$ минимальна.

**Шаг 1. Определяем предсказание**

```php
function predict(array $x, float $w): array {
    return array_map(fn($xi) => $w * $xi, $x);
}
```

**Шаг 2. Определяем MSE**

```php
function mse(array $y, array $yHat): float {
    $sum = 0.0;
    $n = count($y);

    if ($n === 0) {
        return 0.0;
    }

    for ($i = 0; $i < $n; $i++) {
        $sum += ($y[$i] - $yHat[$i]) ** 2;
    }

    return $sum / $n;
}
```

**Шаг 3. Обучение "в лоб" – перебор параметра**

```php
$bestW = null;
$bestLoss = INF;

for ($w = 0; $w <= 3; $w += 0.01) {
    $yHat = predict($x, $w);
    $loss = mse($y, $yHat);

    if ($loss < $bestLoss) {
        $bestLoss = $loss;
        $bestW = $w;
    }
}

echo "Best w ≈ $bestW, loss ≈ $bestLoss";
echo "Predict for 5: ≈ " . (5 * $bestW);
```

Результат:

```
Best w ≈ 2.0, loss ≈ 0.0
Predict for 5: ≈ 10
```

#### Что мы только что сделали

Мы:

1. Выбрали параметр $$w$$.
2. Посчитали ошибку.
3. Попробовали другое значение.
4. Нашли то, где ошибка минимальна

Это и есть обучение: без градиентов, без производных, без библиотек.

Просто поиск минимума функции ошибки.

#### Почему это принципиально важно

Читатель буквально видит:

> Обучение модели = поиск такого параметра, при котором loss минимальна.

Не "магия", не "чёрный ящик", а обычная оптимизация.

#### Как это выглядит в реальной библиотеке (Rubix ML)

Теперь покажем то же самое, но через библиотеку. Используем уже хорошо известную нам библиотеку Rubix ML.

```php
use Rubix\ML\Datasets\Labeled;
use Rubix\ML\Datasets\Unlabeled;
use Rubix\ML\Regressors\Ridge;

$samples = [[1], [2], [3], [4]];
$labels = [2, 4, 6, 8];

$dataset = new Labeled($samples, $labels);

$model = new Ridge(1e-6);
$model->train($dataset);

$predictions = $model->predict(new Unlabeled([[5]]));

$coefficients = $model->coefficients();
$bestW = number_format($coefficients[0], 2, '.', '');
$bestLoss = number_format($model->bias(), 2, '.', '');

echo "Best w ≈ $bestW, loss ≈ $bestLoss";
echo "Predict for 5: ≈ "  . $predictions[0];
```

Результат будет близок к:

```
Best w ≈ 2.00, loss ≈ 0.00
Predict for 5: ≈ 9.9999990
```

**Что происходит "под капотом"**

Метод `train()`:

* решает задачу наименьших квадратов
* находит такие параметры, которые минимизируют MSE
* получает коэффициент, близкий к 2

То же самое, что мы сделали вручную. Просто быстрее и математически аккуратнее.

Теперь становится очевидно:

* loss-функция – это не просто метрика качества после обучения, она задаёт цель оптимизации
* цель оптимизации определяет, какие параметры будут считаться "лучшими"
* следовательно, выбор loss напрямую влияет на итоговую модель

В общем случае, если бы мы минимизировали [MAE](../../../vvedenie/zaklyuchitelnye-materialy/glossarii.md#mae-mean-absolute-error-srednyaya-absolyutnaya-oshibka) вместо [MSE](../../../vvedenie/zaklyuchitelnye-materialy/glossarii.md#mse-mean-squared-error), оптимальные параметры могли бы отличаться (особенно при наличии выбросов).

#### Главный вывод кейса

Обучение модели – это:

> процесс поиска минимума функции ошибки по пространству параметров модели.

Именно поэтому вся теория оптимизации, градиенты и производные существуют – они лишь способы быстрее найти минимум loss, который мы здесь нашли перебором.
