# Пример 4. Batch и стохастический спуск

#### Цель

Понять разницу между двумя подходами к обучению:

* batch gradient descent – обновление по всему датасету
* stochastic gradient descent (SGD) – обновление по одному объекту

Этот разбор показывает важную идею:

обучение может быть гладким и стабильным или шумным и дерганым, и это не ошибка, а осознанный выбор алгоритма.

#### Сценарий

Мы снова используем знакомый простой датасет:

```
x = [1, 2, 3, 4]
y = [2, 4, 6, 8]
```

Истинная зависимость:

$$
y = 2x
$$

Модель:

$$
\hat{y} = w x
$$

Функция ошибки – MSE.

Далее мы запускаем два варианта обучения:

1. Сначала считаем градиент по всем данным сразу (batch)
2. Затем считаем градиент по каждому объекту отдельно (SGD)

#### Реализация на PHP

**Batch Gradient Descent**

```php
$x = [1, 2, 3, 4];
$y = [2, 4, 6, 8];

$w = 0.0;
$lr = 0.1;
$n = count($x);

echo "Batch GD\n";

for ($epoch = 1; $epoch <= 10; $epoch++) {

    $gradient = 0.0;

    for ($i = 0; $i < $n; $i++) {
        $gradient += $x[$i] * (($w * $x[$i]) - $y[$i]);
    }

    $gradient = (2 / $n) * $gradient;
    $w -= $lr * $gradient;

    echo "Epoch $epoch: w = " . round($w, 4) . "\n";
}
```

**Stochastic Gradient Descent**\
В реальных задачах перед каждой эпохой данные обычно перемешиваются (shuffle), чтобы избежать систематических смещений.

```php
$x = [1, 2, 3, 4];
$y = [2, 4, 6, 8];

$w = 0.0;
$lr = 0.1;
$n = count($x);

echo "Stochastic GD\n";

for ($epoch = 1; $epoch <= 10; $epoch++) {

    for ($i = 0; $i < $n; $i++) {
        $gradient = 2 * $x[$i] * (($w * $x[$i]) - $y[$i]);
        $w -= $lr * $gradient;
    }

    echo "Epoch $epoch: w = " . round($w, 4) . "\n";
}
```

#### Что происходит во время обучения

Если сравнить вывод двух частей программы, сразу видно различие в поведении.

**Batch gradient descent – гладкое движение**

В batch-режиме на каждой эпохе:

* используется весь датасет
* считается усредненный градиент
* делается одно обновление

В результате:

* значения $$w$$ меняются плавно
* траектория выглядит предсказуемой
* ошибка уменьшается стабильно

Это классическая формулировка градиентного спуска. В современных задачах чаще используется mini-batch градиентный спуск, который является компромиссом между batch и SGD.

**Stochastic gradient descent – шумное движение**

В SGD обновление происходит после каждого объекта:

* взяли одну точку
* посчитали градиент
* сразу обновили параметр

Если не перемешивать данные, порядок объектов может влиять на траекторию обучения.

Из-за этого:

* каждое обновление "тянет" параметр в свою сторону
* траектория становится рваной
* значения $$w$$ могут колебаться

Вместо гладкой линии получается "дрожащая" траектория.

#### Ключевое наблюдение

Самое важное, что нужно увидеть:

SGD шумит – но это не баг.

Шум возникает потому, что каждая точка дает свою оценку градиента. Формально это означает, что градиент по одному объекту является несмещенной, но высокодисперсной оценкой истинного градиента.

Это приближенная, а не точная информация о направлении минимума.

#### Интуитивное объяснение

Представьте, что вы ищете минимум функции.

* Batch GD – это как если бы вы каждый раз смотрели на всю карту целиком
* SGD – это как если бы вы ориентировались только по одному наблюдению за раз

В первом случае движение аккуратное, но требует больше вычислений.

Во втором – движение хаотичное, но реакции происходят быстрее.

#### Почему SGD часто быстрее на практике

Хотя SGD выглядит менее стабильным, у него есть важное преимущество:

* он делает более частые обновления параметров
* быстрее реагирует на структуру данных
* иногда помогает выходить из сложных участков (например, плато или локальных минимумов)

Иногда именно шум помогает выйти из плохих областей функции ошибки. При этом важно понимать: SGD не всегда требует меньше шагов для сходимости, но часто быстрее достигает хорошего решения по реальному времени (wall-clock time), особенно на больших данных.

#### Практическое значение

В реальных задачах почти всегда используются вариации SGD:

* mini-batch gradient descent (компромисс между batch и SGD)
* momentum
* Adam и другие адаптивные методы

Полный batch используется реже, особенно на больших данных, потому что:

* он вычислительно дорог
* медленно обновляет параметры

#### Выводы

Этот пример показывает фундаментальное различие двух подходов к обучению.

Batch gradient descent дает:

* стабильность
* предсказуемость
* гладкую траекторию

Stochastic gradient descent дает:

* скорость реакции
* шум
* более "живое" поведение

Главное понимание: SGD использует не точный градиент, а его шумную оценку, и этот шум – не ошибка, а полезное свойство
