# Пример 2. Влияние learning rate

#### Цель

Понять, как параметр learning rate влияет на процесс обучения модели.

Мы увидим, что происходит с параметром модели при слишком маленьком, разумном и слишком большом значении learning rate.

Главная задача этого разбора – показать, что значение learning rate определяет характер движения по ландшафту ошибки.

Один и тот же алгоритм может сходиться быстро, сходиться медленно или вообще не сходиться – только из-за значения learning rate.

#### Сценарий

Мы используем тот же простой набор данных, что и в предыдущем примере:

```
x = [1, 2, 3, 4]
y = [2, 4, 6, 8]
```

Истинная зависимость снова равна:  $$y = 2x$$.

Модель:

$$
\hat{y} = w \cdot x
$$

Функция ошибки – среднеквадратичная (MSE).

Как и в предыдущем примере, метод оптимизации – [batch gradient descent](../../../vvedenie/5.-glossarii.md#batch-gradient-descent). Единственное отличие – мы будем запускать обучение с разными значениями learning rate.

Мы проверим три сценария:

* очень маленький шаг
* разумный шаг
* слишком большой шаг

#### Реализация на PHP

```php
function train(float $lr, array $x, array $y): void {
    $w = 0.0;
    $n = count($x);

    echo "Learning rate = $lr\n";

    for ($epoch = 1; $epoch <= 10; $epoch++) {

        $gradient = 0.0;

        for ($i = 0; $i < $n; $i++) {
            $error = ($w * $x[$i]) - $y[$i];
            $gradient += $x[$i] * $error;
        }

        $gradient = (2 / $n) * $gradient;
        $w -= $lr * $gradient;

        echo "Epoch $epoch: w = " . round($w, 4) . "\n";
    }
}

// Простые обучающие данные, где целевое соотношение y = 2x.
$x = [1, 2, 3, 4];
$y = [2, 4, 6, 8];

train(0.01, $x, $y);   // слишком медленно
train(0.1, $x, $y);    // нормально
train(1.0, $x, $y);    // расходимость
```

Каждый запуск обучения начинается с одинаковых условий:

* одинаковые данные
* одинаковая модель
* одинаковое начальное значение параметра

Единственное отличие – learning rate.

#### Что происходит во время обучения

Если внимательно посмотреть на вывод программы, становится ясно, насколько сильно learning rate влияет на поведение модели.

**Слишком маленький learning rate**

При значении

$$
\eta = 0.01
$$

модель движется в правильном направлении, но делает это очень медленно.

```
Epoch 10: w = 1.6063
```

Параметр $$w$$ постепенно приближается к оптимальному значению, но за 10 эпох не успевает даже приблизиться к оптимальному значению $$w = 2$$. Это похоже на спуск с горы очень маленькими шагами. Вы движетесь вниз, но путь занимает слишком много времени.

**Разумный learning rate**

При значении

$$
\eta = 0.1
$$

параметр начинает быстро приближаться к оптимуму.

```
Epoch 10: w = 1.998
```

После нескольких эпох значение w становится близким к 2. Ошибка уменьшается быстро и стабильно. Это и есть нормальный режим работы градиентного спуска. Шаги достаточно большие, чтобы обучение происходило быстро, но не настолько большие, чтобы система потеряла устойчивость.

**Слишком большой learning rate**

При значении

$$
\eta = 1.0
$$

начинается совсем другое поведение.

```
Epoch 10: w = -578509309950
```

Параметр делает слишком большой шаг и может перелететь через минимум. Следующее обновление уводит его далеко в противоположную сторону. После этого процесс повторяется. Вместо движения к минимуму модель начинает прыгать вокруг него. Иногда такие колебания даже усиливаются, из-за чего ошибка начинает расти. В этом случае говорят, что обучение расходится.

#### Интерпретация

Learning rate можно представить как длину шага при спуске по холму.

* слишком маленький шаг – вы движетесь очень медленно;
* нормальный шаг – вы быстро и уверенно спускаетесь;
* слишком большой шаг – вы начинаете перескакивать через низину.

Важно понимать, что градиентный спуск сам по себе не гарантирует сходимости при произвольном значении learning rate. Он лишь задает направление движения. Длина шага определяет, будет ли это движение стабильным.

#### Почему это важно на практике

Когда модель "не обучается", очень часто причина не в данных и не в архитектуре модели, а именно в learning rate.

Слишком маленькое значение приводит к тому, что обучение выглядит практически неподвижным: параметры меняются очень медленно.

Слишком большое значение делает процесс нестабильным.

Поэтому в реальных системах подбор learning rate является одной из первых задач при обучении модели.

#### Выводы

Этот пример показывает несколько важных свойств градиентного спуска.

Во-первых, learning rate управляет скоростью движения и устойчивостью обучения на ландшафте ошибки. Один и тот же алгоритм может вести себя совершенно по-разному в зависимости от размера шага.

Во-вторых, слишком большой шаг может полностью разрушить обучение. Вместо спуска модель начинает перескакивать минимум.

И наконец, становится ясно, почему выбор learning rate считается одним из самых чувствительных параметров в машинном обучении. Один и тот же алгоритм, один и тот же датасет, одна и та же модель – но три совершенно разных поведения системы.
