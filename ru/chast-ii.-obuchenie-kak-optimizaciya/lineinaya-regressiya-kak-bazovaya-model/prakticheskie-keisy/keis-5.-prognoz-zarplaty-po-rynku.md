# Кейс 5. Прогноз зарплаты по рынку

#### Суть кейса

Один из самых практических сценариев применения регрессионных моделей – анализ рынка труда. Компании хотят понимать, какую зарплату ожидать кандидатам, а кандидаты – сколько они реально стоят на рынке.

В HR-tech продуктах и аналитике вакансий часто строят модели, которые оценивают ожидаемый уровень зарплаты на основе характеристик кандидата и условий работы. Такие модели используются в системах подбора персонала, salary benchmarks, аналитике рынка и даже в системах автоматической публикации вакансий.

Этот кейс демонстрирует, как с помощью простой регрессионной модели можно построить приближённую оценку рыночной зарплаты, а также поднимает важную тему – [bias](../../../vvedenie/zaklyuchitelnye-materialy/glossarii.md#bias-smeshenie) и fairness в данных.

#### Признаки

В нашем примере используются четыре базовых признака:

* опыт работы (в годах)
* количество технологий, которыми владеет разработчик
* размер компании
* удалённая работа (0 или 1)

Размер компании можно представить как числовую категорию, например:

* 1 – стартап
* 2 – средняя компания
* 3 – крупная компания

Это упрощённое представление. В реальных моделях категориальные признаки обычно кодируются с помощью one-hot encoding, чтобы модель не воспринимала категории как линейную шкалу.

В нашем примере набор признаков намеренно упрощён. Он не описывает человека полностью, но хорошо демонстрирует, как модель начинает выявлять статистические закономерности рынка.

#### Цель

Задача модели – оценить типичный уровень зарплаты, который можно ожидать на рынке при данных параметрах.

Почему именно медианная зарплата? Потому что зарплатные данные обычно имеют сильный перекос: небольшое количество очень высоких зарплат может сильно искажать среднее значение. Медиана гораздо лучше отражает типичную рыночную ситуацию.

В реальных задачах для прямого моделирования медианы часто используют quantile regression. В этом примере мы применяем обычную линейную регрессию, чтобы упростить объяснение механики модели.

#### Почему это хороший финальный кейс

Этот пример особенно полезен для завершения темы линейной регрессии, потому что он затрагивает сразу несколько важных аспектов.

Во-первых, это чувствительные данные. В отличие от CPU или чека интернет-магазина, зарплаты напрямую влияют на людей и карьеру.

Во-вторых, этот кейс позволяет обсудить предвзятость в данных. Если исторические данные содержат перекосы, модель неизбежно их воспроизведёт.

В-третьих, коэффициенты модели здесь особенно легко интерпретировать: каждый вес буквально показывает сколько в среднем "стоит" дополнительный год опыта или новая технология.

Таким образом, это не только технический, но и социальный пример применения ML.

#### Подготовка данных

Предположим, что мы собрали агрегированные данные с рынка труда. Каждый объект – это условный профиль кандидата, а целевая переменная – медианная зарплата по таким параметрам.

```php
use Rubix\ML\Datasets\Labeled;

$samples = [
    // опыт, технологии, размер компании, удалёнка
    [1, 2, 1, 1],
    [3, 4, 2, 1],
    [5, 6, 3, 0],
    [7, 8, 3, 1],
    [10, 10, 3, 1],
];

// Target: salary in USD
$labels = [
    1500,
    2800,
    4500,
    6200,
    8000,
];

$dataset = Labeled::build($samples, $labels);
```

Здесь данные сильно упрощены, но они хорошо иллюстрируют структуру задачи.

#### Обучение модели

Для обучения используем линейную регрессию с регуляризацией из библиотеки Rubix ML. Регуляризация помогает избежать ситуации, когда модель начинает слишком сильно полагаться на один признак.

```php
use Rubix\ML\Regressors\Ridge;

$model = new Ridge(1.0);
$model->train($dataset);
```

После обучения модель будет оценивать вклад каждого признака в итоговую зарплату.

#### Предсказание

Теперь попробуем оценить зарплату кандидата со следующими характеристиками:

* 4 года опыта
* 5 технологий
* средняя компания
* удалённая работа

```php
use Rubix\ML\Datasets\Unlabeled;

$candidate = [4, 5, 2, 1];

$unlabeled = new Unlabeled([$candidate]);
$prediction = $model->predict($unlabeled);

$salary = $prediction[0];

echo 'Ожидаемая зарплата:' . round($salary, 2) . PHP_EOL . PHP_EOL;
echo 'Коэффициенты (веса признаков):' . PHP_EOL;
echo '0 - опыт, 1- технологии, 2- размер компании, 3 - удалёнка' . PHP_EOL;
print_r($weights);

echo PHP_EOL . 'Bias (intercept): ' . $bias . PHP_EOL;

// Результат: 
// Ожидаемая зарплата: 3750 USD
// Коэффициенты (веса признаков):
// 0 - опыт, 1- технологии, 2- размер компании, 3 - удалёнка
// Array (
//     [0] => 387.50
//     [1] => 375.00
//     [2] => 37.50
//     [3] => 25.00
// )
// Bias (intercept): 225.00
```

Конечно, это не точная цифра, а оценка на основе статистики рынка, но она может быть полезной для ориентирования.

#### Интерпретация коэффициентов

Одно из главных преимуществ линейной регрессии – прозрачность. Если посмотреть на веса модели, можно буквально прочитать её логику.

Например, модель может показать:

* каждый год опыта добавляет примерно X к зарплате
* каждая дополнительная технология в стеке добавляет примерно Y
* работа в крупной компании увеличивает зарплату на Z
* удалённая работа может коррелировать с небольшим бонусом

Такая интерпретация делает модель понятной не только инженерам, но и HR-командам. Однако такая прозрачность поднимает ещё один важный вопрос – насколько эти зависимости справедливы.

#### Bias и fairness

Но здесь появляется важный вопрос: насколько справедлива такая модель.

Если исторические данные содержат перекосы – например, разные зарплаты для одинаковых специалистов в разных регионах или компаниях – модель автоматически воспроизведёт эти перекосы.

Это ключевой принцип машинного обучения:

> модель не создаёт новую реальность – она воспроизводит статистику данных.

Поэтому при работе с такими задачами важно:

* анализировать источники данных
* проверять распределения
* тестировать модель на разных группах
* использовать ML как инструмент анализа, а не как окончательный суд

#### Ограничения модели

Как и в предыдущих кейсах, линейная регрессия здесь является лишь первой ступенью. Помимо возможных перекосов в данных, модель имеет и структурные ограничения.

Она не учитывает множество факторов, которые влияют на зарплату:

* специализацию (backend, ML, DevOps)
* регион
* уровень компании
* soft skills
* сложность проектов

Тем не менее она позволяет быстро построить рабочую модель рынка, которую можно постепенно улучшать.

#### Выводы

Этот кейс хорошо демонстрирует, как простая регрессионная модель может использоваться для анализа реальных социально-экономических данных.

Мы видим несколько важных уроков:

* Линейная регрессия позволяет быстро построить baseline-модель рынка труда
* Коэффициенты модели дают интерпретируемое объяснение факторов зарплаты
* Качество модели напрямую зависит от качества данных
* В задачах, связанных с людьми, всегда нужно учитывать вопросы bias и fairness

Именно поэтому такие модели полезны не только как инструмент предсказания, но и как инструмент понимания данных и рынка.
