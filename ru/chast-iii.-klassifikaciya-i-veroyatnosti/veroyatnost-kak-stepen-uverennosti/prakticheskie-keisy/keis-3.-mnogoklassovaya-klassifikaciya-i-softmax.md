# Кейс 3. Многоклассовая классификация и softmax

#### Цель кейса

Цель этого кейса – показать, как в задачах многоклассовой классификации модель выражает распределение уверенности между несколькими гипотезами, а не просто выбирает один "правильный" ответ.

Мы рассмотрим ситуацию, где входящее письмо может относиться к нескольким категориям: например, обычное письмо, рекламная рассылка или спам. Модель оценивает вероятность каждого класса, а не только выбирает один наиболее вероятный вариант. Эти вероятности позволяют системе принимать более гибкие решения.

Ключевая идея: [softmax](../../../vvedenie/zaklyuchitelnye-materialy/glossarii.md#softmax) превращает оценки модели в распределение вероятностей, сумма которых равна 1.

#### Сценарий

Представим почтовый сервис, который автоматически сортирует письма. Каждое письмо может относиться к одной из нескольких категорий:

* normal – обычное письмо
* promo – рекламная рассылка
* spam – спам

Для пользователя важно не только определить наиболее вероятный класс, но и понимать насколько уверена модель. Иногда письмо может иметь признаки, характерные и для рекламы, и для спама, и модель должна выразить эту неопределенность.

Многоклассовые вероятности позволяют:

* анализировать уверенность модели
* строить разные правила обработки писем
* реализовывать более гибкую сортировку

#### Подготовка модели

В библиотеке RubixML для многоклассовых вероятностей используется SoftmaxClassifier. Он берет базовый алгоритм классификации и преобразует его выходные оценки в вероятностное распределение с помощью функции softmax.

В качестве базового алгоритма используем k-ближайших соседей.

```php
use Rubix\ML\Datasets\Labeled;
use Rubix\ML\Datasets\Unlabeled;
use Rubix\ML\Classifiers\SoftmaxClassifier;

/* Признаки письма: [длина темы, количество ссылок] */
$samples = [
    // normal
    [4, 0],
    [6, 1],
    // promo
    [10, 3],
    [12, 4],
    // spam
    [15, 7],
    [18, 9],
];

$labels = [
    'normal',
    'normal',
    'promo',
    'promo',
    'spam',
    'spam',
];

$dataset = new Labeled($samples, $labels);

$model = new SoftmaxClassifier();
$model->train($dataset);
```

Здесь происходит следующее:

* SoftmaxClassifier оценивает, насколько новое письмо похоже на примеры из обучающего набора на основе заданных признаков
* Затем классификатор применяет функцию softmax, которая нормализует полученные оценки и преобразует их в распределение вероятностей по всем возможным классам (например: normal, promo, spam)

#### Предсказание вероятностей

После обучения модель может вернуть вероятности для нового письма.

```php
/* Новое письмо: длинная тема, несколько ссылок */
$sample = new Unlabeled([[14, 5]]);
$probabilities = $model->proba($sample)[0];

print_r($probabilities);
```

Пример распределения вероятностей может выглядеть так:

```php
[
    [normal] => 0.067174171341534
    [promo] => 0.66731556239214
    [spam] => 0.26551026626633
]
```

Важно заметить несколько вещей:

* каждое значение находится в диапазоне от 0 до 1
* сумма всех вероятностей равна 1
* модель распределяет уверенность между всеми возможными классами

#### Интерпретация результата

В этом примере модель считает наиболее вероятным класс promo, но уверенность не является абсолютной:

* реклама: 46%
* спам: 33%
* обычное письмо: 21%

Это означает, что письмо обладает признаками сразу нескольких категорий. Такая ситуация часто встречается в реальных данных.

Если бы система возвращала только один класс (promo), мы потеряли бы важную информацию о степени сомнения модели.

#### Как используется распределение вероятностей

Наличие распределения вероятностей открывает дополнительные возможности:

1. Можно применять разные правила обработки. Например:

* если spam > 0.8 – переместить в спам
* если promo > 0.4 – отправить в папку "Промо"
* иначе оставить во входящих.

2. Можно показывать пользователю объяснение или индикатор уверенности.
3. Можно анализировать сложные случаи, когда модель сильно сомневается.

Таким образом, вероятности становятся интерфейсом между моделью и системой принятия решений.

#### Почему используется softmax

Функция softmax выполняет две важные задачи:

1. Нормализует оценки модели так, чтобы они стали вероятностями.
2. Подчеркивает различия между классами, усиливая более уверенные оценки.

Формально функция softmax выглядит так:

$$
P(y = i) = \frac{\exp(z_i)}{\sum_{j} \exp(z_j)}
$$

где $$z_i$$ – оценка модели для класса $$i$$.

После применения softmax все значения находятся в диапазоне от 0 до 1 и образуют корректное распределение вероятностей.

#### Типичная ошибка

Распространенная ошибка – использовать только класс с максимальной вероятностью и игнорировать остальные значения.

В этом случае система фактически возвращается к бинарному мышлению: "правильно / неправильно". Однако именно распределение уверенности является одной из самых полезных характеристик современных моделей.

#### Выводы

1. В многоклассовых задачах модель должна выражать уверенность по каждому классу.
2. Softmax преобразует оценки модели в корректное распределение вероятностей.
3. Распределение вероятностей содержит больше информации, чем просто победивший класс.
4. Эти вероятности позволяют системе принимать гибкие и контекстные решения.

Этот кейс показывает, как вероятности становятся мостом между моделью машинного обучения и реальной логикой приложения.
