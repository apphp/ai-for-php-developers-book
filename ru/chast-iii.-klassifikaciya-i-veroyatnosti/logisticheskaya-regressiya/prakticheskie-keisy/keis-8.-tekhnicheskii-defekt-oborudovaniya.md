# Кейс 8. Технический дефект оборудования

#### Цель

Построить модель, которая заранее определяет вероятность поломки оборудования на основе его текущего состояния.

Это позволяет:

* снижать простои
* &#x20;переходить от "реактивного" ремонта к "предиктивному"
* экономить на обслуживании

#### Сценарий

Представим промышленное оборудование (например, насос, сервер или станок).

У нас есть три ключевых признака:

* температура (°C)
* уровень вибрации (условная шкала)
* общее время работы (часы)

Исторические данные показывают (в данном примере используется небольшой иллюстративный датасет):

* 0 → оборудование работает нормально
* 1 → произошла поломка

#### Данные

```php
// температура, уровень вибрации, общее время работы
$samples = [
    [70, 0.1, 1000],
    [95, 0.6, 8000],
    [80, 0.2, 2000],
    [110, 0.9, 12000],
    [75, 0.15, 1500],
    [100, 0.7, 9000],
    [85, 0.3, 3000],
    [108, 0.85, 11000],
];

// 0 = нормальный результат, 1 = риск неудачи.
// Классификаторы RubixML ожидают категориальные метки в виде строк,
// поэтому даже числовые классы ('0', '1') хранятся как строки 
// (в данном примере мы не будем переводить их в слова).
$labels = ['0', '1', '0', '1', '0', '1', '0', '1'];
```

Интуитивно видно:

* высокая температура + сильная вибрация + большой износ → поломка
* низкие значения → нормальная работа

#### Модель

Здесь идеально подходит логистическая регрессия, потому что:

* задача бинарная (есть дефект или нет)
* на выходе получаем вероятность
* легко интерпретировать

#### Реализация на PHP

```php
use Rubix\ML\Classifiers\LogisticRegression;
use Rubix\ML\Datasets\Labeled;
use Rubix\ML\Datasets\Unlabeled;

$samples = [
    [70, 0.1, 1000],
    [95, 0.6, 8000],
    [80, 0.2, 2000],
    [110, 0.9, 12000],
    [75, 0.15, 1500],
    [100, 0.7, 9000],
    [85, 0.3, 3000],
    [108, 0.85, 11000],
];

$labels = ['0', '1', '0', '1', '0', '1', '0', '1'];

$dataset = new Labeled($samples, $labels);

$model = new LogisticRegression();
$model->train($dataset);

// Новый объект оборудования
$equipment = new Unlabeled([[98, 0.5, 8000]]);

$result = $model->predict($equipment);
$probabilities = $model->proba($equipment);

echo "Предсказанная метка (1 или 0): " . $result[0] . "\n";
echo "Вероятность поломки: " . $probabilities[0]['1'] . "\n";
echo "Решение: ";
if ($result[0] == '1') {
    echo 'Риск поломки';
} else {
    echo 'Нормально';
}

// Результат:
// Предсказанная метка (1 или 0): 1
// Вероятность поломки: 1
// Решение: Риск поломки
```

#### Интерпретация результата

Если модель возвращает:  `1`

это означает:

* оборудование находится в зоне риска
* вероятна поломка

Если: `0`

* всё в норме

#### Что происходит внутри

Модель строит зависимость:

$$
P(\text{поломка}) = \sigma(w_1 \cdot \text{температура} + w_2 \cdot \text{вибрация} + w_3 \cdot \text{время} + b)
$$

Где:

* σ (сигмоида) переводит значение в вероятность (0–1)
* веса (w) показывают важность каждого признака

На практике часто наблюдается:

* вибрация является одним из наиболее информативных признаков
* время работы усиливает риск
* влияние температуры может быть критичным при перегреве

#### Практическое использование

Такую модель можно встроить в:

* IoT-систему (Интернет вещей) мониторинга
* SCADA (Supervisory Control and Data Acquisition) / промышленный контроль&#x20;
* серверный мониторинг
* Система прогнозирующего технического обслуживания (predictive maintenance)

Пример логики:

```php
if ($model->predict($equipment)[0] == '1') {
    echo "⚠️ Возможна поломка – требуется обслуживание";
}
```

#### Ограничения

Важно понимать:

* данных мало → модель примитивная
* нет временной динамики (важно для реальных систем)
* признаки не нормализованы (для логистической регрессии это особенно критично из-за разного масштаба признаков)
* нет учета резких скачков

В реальности добавляют:

* производные (изменение температуры)
* частотный анализ вибрации
* историю (time series)

#### Как улучшить

* добавить больше данных
* использовать стандартизацию признаков
* применить [Random Forest](https://en.wikipedia.org/wiki/Random_forest) или [Gradient Boosting](https://en.wikipedia.org/wiki/Gradient_boosting)
* перейти к временным моделям&#x20;
