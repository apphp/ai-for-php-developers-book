# Кейс 3. Числовые признаки (Gaussian Naive Bayes)

До этого момента мы работали с категориальными признаками: слово есть или нет, признак равен 0 или 1. Но в реальных задачах данные чаще числовые – время, количество, температура, расстояние.

В этом кейсе мы рассмотрим, как наивный Байес работает с такими признаками. Для этого используется вариант модели, называемый [Gaussian Naive Bayes](../../../vvedenie/5.-glossarii.md#gaussian-naive-bayes), где предполагается, что значения признаков внутри каждого класса распределены нормально.

#### Цель кейса

Показать, как:

* наивный Байес работает с числовыми признаками
* вместо частот используются параметры распределения (среднее и дисперсия)
* вероятности вычисляются через нормальное распределение
* модель сохраняет ту же логику: произведение вероятностей → лог-сумма

#### Сценарий

Мы классифицируем пользователя как:

* active – активно взаимодействует с сайтом
* inactive – практически не проявляет активности

Используем два числовых признака:

* время на сайте (в секундах)
* количество кликов

Интуитивно:

* активные пользователи проводят больше времени и делают больше действий
* неактивные – наоборот

#### Реализация на чиcтом PHP

#### Обучающая выборка

```php
$samples = [
    [120, 10],
    [130, 12],
    [20,  1],
    [30,  2],
];

$labels = ['active', 'active', 'inactive', 'inactive'];
```

Здесь уже нельзя просто считать частоты. Значения непрерывные, и нам нужно оценить распределение.

#### Шаг 1. Оценка параметров распределения

Для каждого класса и каждого признака мы считаем:

* среднее (mean)
* дисперсию (variance)

Пример для одного признака:

```php
function mean($values) {
    return array_sum($values) / count($values);
}

// Здесь используется оценка дисперсии генеральной совокупности 
// (в ML это допустимое упрощение).
function variance($values, $mean) {
    $sum = 0;
    foreach ($values as $v) {
        $sum += pow($v - $mean, 2);
    }
    return $sum / count($values);
}
```

Далее группируем данные по классам:

```php
$grouped = [];

foreach ($samples as $i => $sample) {
    $class = $labels[$i];
    $grouped[$class][] = $sample;
}
```

Считаем параметры:

```php
$stats = [];

foreach ($grouped as $class => $rows) {
    $features = array_map(null, ...$rows); // транспонирование

    foreach ($features as $i => $values) {
        $m = mean($values);
        $v = variance($values, $m);

        $stats[$class][$i] = [
            'mean' => $m,
            'variance' => $v ?: 1e-6, // защита от нуля
        ];
    }
}
```

#### Шаг 2. Функция плотности нормального распределения

Реализуем функции плотности нормального (гауссовского) распределения

$$
f(x)=\frac{1}{\sqrt{2\pi\sigma^2}} \cdot \exp\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)
$$

```php
function gaussian($x, $mean, $variance) {
    return (1 / sqrt(2 * pi() * $variance)) * exp(-pow($x - $mean, 2) / (2 * $variance));
}
```

Это плотность вероятности $$f(x \mid C)$$ (в модели она используется как $$P(x \mid C)$$ для сравнения классов).

#### Шаг 3. Классификация нового пользователя

Допустим, пользователь:

* провел 100 секунд
* сделал 9 кликов

```php
$input = [100, 9];
$classCounts = array_count_values($labels);
$total = count($labels);

$scores = [];

foreach ($stats as $class => $features) {
    $logProb = log($classCounts[$class] / $total);

    foreach ($features as $i => $params) {
        $prob = gaussian($input[$i], $params['mean'], $params['variance']);
        // защита от log(0)
        $prob = max($prob, 1e-12);
        $logProb += log($prob);
    }

    $scores[$class] = $logProb;
}

arsort($scores);
print_r($scores);

// Результат:
// Array (
//   [active] => -18.640462159403
//   [inactive] => -228.44731497884
// )
```

Сравниваем:

* active = -18.64
* inactive = -228.447

-18.64 > -228.447 → модель выбирает **active**

Модель:

1. Берет априорную вероятность класса
2. Для каждого признака считает вероятность через нормальное распределение
3. Складывает логарифмы
4. Выбирает лучший класс

#### Что здесь важно понять

#### 1. Частоты заменяются распределением

В категориальном случае: $$P(word \mid class)$$

В числовом: $$P(x \mid class)$$ через нормальное распределение

Это ключевое отличие Gaussian Naive Bayes.

#### 2. Модель остается той же

Несмотря на другую формулу:

* признаки все так же независимы
* перемножаются правдоподобия (likelihoods)
* используется лог-суммирование
* выбирается максимум

Меняется только способ оценки вероятностей.

#### 3. Почему это работает

Даже если реальные данные не идеально нормальны:

* распределение часто достаточно близко
* ошибки оценки не критичны
* модель правильно ранжирует классы

Это тот же принцип, что и в предыдущих кейсах: важна не абсолютная точность вероятностей, а относительное сравнение. Модель может неточно оценивать сами вероятности, но при этом хорошо разделять классы.

#### Ограничения

* предположение нормальности не всегда выполняется
* чувствительность к выбросам
* признаки могут быть зависимы
* при маленьких данных оценки нестабильны

Тем не менее, модель часто дает хороший baseline.

<summary>Кейс 3. Полный пример кода на чистом PHP</summary>

```php
// Числовой набор данных: каждый образец содержит 2 числовых признака.
$samples = [
    [120, 10],
    [130, 12],
    [20, 1],
    [30, 2],
];

// Метка класса для каждого образца.
$labels = ['active', 'active', 'inactive', 'inactive'];

// Оценки среднего значения и дисперсии (дисперсия генеральной совокупности).
function mean(array $values): float {
    return array_sum($values) / count($values);
}

function variance(array $values, float $mean): float {
    $sum = 0.0;
    foreach ($values as $v) {
        $sum += pow($v - $mean, 2);
    }

    return $sum / count($values);
}

// Гауссова функция плотности вероятности (вероятность для числового признака).
function gaussian(float $x, float $mean, float $variance): float {
    return (1 / sqrt(2 * pi() * $variance)) * exp(-pow($x - $mean, 2) / (2 * $variance));
}

// Примеры группового обучения по классам.
$grouped = [];

foreach ($samples as $i => $sample) {
    $class = $labels[$i];
    $grouped[$class][] = $sample;
}

// Статистика по классам и признакам: [class][featureIndex] => ['mean' => ..., 'variance' => ...]
$stats = [];

foreach ($grouped as $class => $rows) {
    // Чтобы получить массивы значений, отображающие отдельные признаки, 
    // преобразуем строки в столбцы.
    $features = array_map(null, ...$rows);

    foreach ($features as $i => $values) {
        $m = mean($values);
        $v = variance($values, $m);

        $stats[$class][$i] = [
            'mean'     => $m,
            'variance' => $v ?: 1e-6,
        ];
    }
}

// Новый числовой образец для классификации.
$input = [100, 9];

// Классовые априорные значения P(класс) на основе частоты встречаемости меток.
$classCounts = array_count_values($labels);
$total = count($labels);

// Вычисляем логарифмическую оценку для каждого класса.
$scores = [];

foreach ($stats as $class => $features) {
    // Начинаем с логарифма априорной вероятности log P(класс).
    $logProb = log($classCounts[$class] / $total);

    foreach ($features as $i => $params) {
        // Складываем логарифм гауссовой функции правдоподобия для каждого признака.
        $prob = gaussian($input[$i], $params['mean'], $params['variance']);
        $logProb += log($prob);
    }

    $scores[$class] = $logProb;
}

// Сортируем классы по оценке (наивысшая / наиболее близкая к 0 оценка является прогнозом).
arsort($scores);
print_r($scores);
```

#### Та же задача с использованием RubixML

```php
use Rubix\ML\Classifiers\GaussianNB;
use Rubix\ML\Datasets\Labeled;

$samples = [
    [120, 10],
    [130, 12],
    [20,  1],
    [30,  2],
];

$labels = ['active', 'active', 'inactive', 'inactive'];

$dataset = new Labeled($samples, $labels);

$model = new GaussianNB();
$model->train($dataset);

$prediction = $model->predict([[100, 9]]);
print_r($prediction);

// Результат:
// Array (
//    [0] => active
// )
```

Как видите, результат такой же как и в примере с чистым PHP.

RubixML автоматически:

* оценивает средние и дисперсии
* применяет нормальное распределение
* считает лог-вероятности
* выбирает класс

#### Вывод

Этот кейс завершает картину наивного Байеса.

Он показывает, что модель:

* не ограничивается бинарными или категориальными данными
* легко расширяется на числовые признаки
* сохраняет ту же базовую логику независимо от типа данных

Главное изменение – не в алгоритме, а в том, как оценивается вероятность признака.

Именно эта гибкость делает наивный Байес универсальным инструментом: от текстов до числовых метрик поведения пользователей.
